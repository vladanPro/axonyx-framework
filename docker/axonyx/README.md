# Axonyx Docker Image

This folder defines the first beta Axonyx demo image.

It is intended to be built as:

```bash
docker build -t axonyx:0.1.87 -f docker/axonyx/Dockerfile .
```

Run it with:

```bash
docker run --rm -p 3000:3000 axonyx:0.1.87
```

After publishing to Docker Hub:

```bash
docker run --rm -p 3000:3000 vladanpro2/axonyx:0.1.87
```

Then open:

```text
http://localhost:3000
```

## What The Image Contains

- Rust `1.95`
- `create-axonyx 0.1.57`
- `cargo-axonyx 0.1.87`
- a generated Axonyx docs starter app
- prebuilt Axonyx output from `cargo ax build --clean`
- OCI image labels for registry metadata
- Docker healthcheck against `/__axonyx/health`
- startup output that prints the local demo URL and package versions

The container starts the generated docs app with:

```bash
cargo ax run start --host 0.0.0.0 --port 3000
```

On startup it prints:

```text
Axonyx demo is running.

Open: http://localhost:3000
Template: docs
CLI: cargo-axonyx 0.1.87
Runtime: axonyx-runtime 0.1.48
```

If a host provides `PORT`, the image uses it:

```bash
docker run --rm -e PORT=8080 -p 8080:8080 axonyx:0.1.87
```

## Build Args

```bash
docker build \
  --build-arg CARGO_AXONYX_VERSION=0.1.87 \
  --build-arg CREATE_AXONYX_VERSION=0.1.57 \
  --build-arg AXONYX_RUNTIME_VERSION=0.1.48 \
  -t axonyx:0.1.87 \
  -f docker/axonyx/Dockerfile .
```

## Future Split

This first image is intentionally a batteries-included demo image.

Later we should split it into:

- `axonyx/cli:<version>` for build/dev images
- `axonyx/runtime:<version>` for slim production runtime images
- app-level Dockerfile templates generated by `create-axonyx`
